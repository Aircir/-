# 大学生竞赛智能咨询系统 PRD

## 1. 文档信息

- 产品名称：大学生竞赛智能咨询系统
- 文档类型：产品需求文档（PRD）
- 版本：V1.0
- 适用阶段：毕业设计答辩演示版 / 可运行的一期版本
- 目标终端：电脑网页端

## 2. 产品概述

大学生在寻找竞赛、理解参赛要求、比较赛事差异和选择适合自己的竞赛时，通常会遇到信息分散、检索效率低、规则理解成本高的问题。传统竞赛信息平台多数仅提供静态信息展示，缺少智能咨询和自然语言辅助操作能力。

本系统面向上述问题，建设一个集竞赛检索、竞赛详情查看、智能咨询、SSE 流式问答、自然语言导航、记录沉淀与后台竞赛管理于一体的 Web 系统。系统重点服务毕业设计答辩演示，同时尽量具备实际使用价值。

## 3. 产品目标

### 3.1 业务目标

- 为学生提供统一的竞赛信息查询入口
- 提供基于竞赛事实信息的智能咨询能力
- 降低学生理解竞赛规则和选择竞赛的门槛
- 为管理员提供基础竞赛数据维护能力
- 为系统演示提供完整、清晰、可操作的业务闭环

### 3.2 一期目标

- 完成学生端核心查询与咨询流程
- 完成管理员端竞赛分类和竞赛信息维护
- 支持连续对话与 SSE 流式回答
- 支持基础自然语言导航
- 支持咨询记录、检索记录、导航记录与基础统计展示

### 3.3 非目标

- 不引入教师角色
- 不将 Excel / CSV 批量导入作为一期刚需
- 不做复杂用户管理和封禁流程
- 不做复杂图表分析系统
- 不做移动端适配优先设计

## 4. 用户角色

### 4.1 学生用户

用户特征：

- 关注如何快速找到目标竞赛
- 关注竞赛是否适合自己的专业和年级
- 关注竞赛区别、准备建议和参赛建议
- 需要更自然的交互方式，而不想只依赖传统筛选

核心诉求：

- 浏览和筛选竞赛
- 查看竞赛详情
- 获得 AI 问答和通用建议
- 支持连续追问
- 查看自己的历史记录
- 查看全局热门统计

### 4.2 管理员

用户特征：

- 负责竞赛基础数据维护
- 关注前台信息准确性
- 需要一个简单直接的后台操作入口

核心诉求：

- 管理竞赛分类
- 管理竞赛信息
- 保证前台竞赛数据可用

## 5. 使用场景

### 场景 1：学生查找适合自己的竞赛

学生进入首页后，通过专业、赛事级别和报名状态进行筛选，快速找到适合自己的竞赛，并进入详情页查看参赛要求。

### 场景 2：学生对竞赛存在疑问

学生查看竞赛详情后，进入智能咨询页，询问“这个竞赛适合我吗”“需要准备什么”“和另外一个竞赛有什么区别”，系统结合竞赛数据生成回答，并允许连续追问。

### 场景 3：学生使用自然语言快捷操作

学生在首页输入“打开竞赛列表页”或“搜索人工智能类竞赛”，系统自动识别意图并完成跳转或筛选。

### 场景 4：管理员维护数据

管理员登录后台，新增竞赛分类、录入竞赛信息、修改报名时间或竞赛简介，保障前台数据准确。

## 7. 信息架构与页面清单

一期系统采用 6 页结构。

### 7.1 登录页

页面目标：

- 作为系统统一入口
- 根据角色进入不同业务区域

页面内容：

- 系统名称与简介
- 用户名输入框
- 密码输入框
- 登录按钮
- 角色识别后的跳转逻辑提示

主要操作：

- 学生登录后进入首页 / 竞赛检索页
- 管理员登录后进入后台管理页

### 7.2 首页 / 竞赛检索页

页面目标：

- 承担首页展示与核心检索入口
- 提供自然语言导航入口

页面内容：

- 顶部导航栏
- 系统简介区
- 自然语言导航输入框
- 检索筛选区
- 竞赛列表区
- 热门竞赛或推荐竞赛区

筛选项：

- 关键词
- 专业
- 赛事级别
- 报名状态
- 竞赛类别
- 年级

主要操作：

- 浏览竞赛列表
- 输入关键词搜索
- 按条件筛选
- 触发自然语言导航
- 点击竞赛进入详情页

### 7.3 竞赛详情页

页面目标：

- 展示单个竞赛的完整信息
- 承接从检索到咨询的过渡

页面内容：

- 竞赛名称
- 竞赛类别
- 赛事级别
- 适合专业
- 适合年级
- 报名开始时间
- 报名结束时间
- 官网链接
- 竞赛简介
- 参赛要求
- 去智能咨询按钮

主要操作：

- 查看竞赛完整信息
- 跳转到智能咨询页
- 返回竞赛检索页

### 7.4 智能咨询页

页面目标：

- 提供围绕竞赛的问答和建议能力
- 展示系统智能化特色

页面内容：

- 提问输入框
- 会话消息区
- 普通问答按钮
- 流式问答按钮
- 示例问题区
- 当前竞赛上下文提示区

主要操作：

- 输入问题发起咨询
- 选择普通返回或 SSE 流式返回
- 继续追问形成连续对话
- 查看问答历史消息

### 7.5 记录与统计页

页面目标：

- 展示用户历史记录
- 展示平台基础统计

页面内容：

- 咨询记录列表
- 检索记录列表
- 导航记录列表
- 统计卡片或统计列表

统计项：

- 咨询总次数
- 热门问题关键词
- 热门竞赛类别
- 常用导航指令
- 活跃时间段

主要操作：

- 查看自己的历史咨询记录
- 查看自己的检索和导航记录
- 查看全局热门统计

### 7.6 后台管理页

页面目标：

- 提供竞赛基础数据维护能力

页面内容：

- 左侧菜单或顶部标签
- 竞赛分类管理区域
- 竞赛信息管理区域

分类管理功能：

- 新增分类
- 编辑分类
- 删除分类
- 查看分类列表

竞赛信息管理功能：

- 新增竞赛
- 编辑竞赛
- 删除竞赛
- 查看竞赛列表

## 8. 核心流程

### 8.1 学生主流程

1. 用户登录系统
2. 进入首页 / 竞赛检索页
3. 浏览或筛选竞赛
4. 点击进入竞赛详情页
5. 阅读竞赛信息
6. 跳转到智能咨询页提问
7. 获取普通回答或流式回答
8. 系统保存咨询记录
9. 用户进入记录与统计页查看历史记录和热门统计

### 8.2 自然语言导航流程

1. 用户在首页输入导航指令
2. 系统对输入做预处理
3. 系统进行规则匹配
4. 返回结构化导航结果
5. 前端执行页面跳转或自动填充筛选

一期优先支持：

- 打开竞赛列表页
- 搜索人工智能类竞赛

### 8.3 管理员主流程

1. 管理员登录系统
2. 进入后台管理页
3. 新增或修改竞赛分类
4. 新增或修改竞赛信息
5. 保存后前台可立即查询更新内容

## 9. 功能需求

### 9.1 用户与登录模块

功能说明：

- 系统提供统一登录入口
- 支持学生和管理员两类账号
- 登录后根据角色跳转页面

功能点：

- 用户名密码校验
- 登录状态保持
- 退出登录
- 角色判断

一期约束：

- 账号可预置
- 注册不是一期核心展示功能

### 9.2 竞赛检索模块

功能说明：

- 为用户提供基于列表和条件筛选的检索能力

功能点：

- 竞赛列表展示
- 关键词搜索
- 按专业筛选
- 按赛事级别筛选
- 按报名状态筛选
- 按类别筛选
- 按年级筛选
- 组合查询
- 空结果提示
- 检索记录保存

输入：

- 关键词
- 专业
- 赛事级别
- 报名状态
- 类别
- 年级

输出：

- 竞赛列表
- 总结果数
- 空状态提示

### 9.3 竞赛详情模块

功能说明：

- 展示竞赛的结构化详细信息

功能点：

- 查询单个竞赛详情
- 展示报名时间
- 展示官网链接
- 展示竞赛简介与参赛要求
- 提供咨询入口

### 9.4 智能咨询模块

功能说明：

- 系统围绕竞赛事实信息和用户问题生成回答
- 在必要时提供通用建议

功能点：

- 问题输入
- 问题预处理
- 关键词提取
- 问题类型识别
- 竞赛事实检索
- 提示词构造
- 模型调用
- 普通问答
- 流式问答
- 连续对话
- 咨询记录保存

问题类型示例：

- 解释类
- 推荐类
- 对比类
- 准备建议类

回答约束：

- 尽量优先基于系统内竞赛数据回答
- 可补充适度通用建议
- 回答应清晰、简洁、易理解

### 9.5 SSE 流式输出模块

功能说明：

- 为智能咨询提供增量输出能力

功能点：

- 建立 SSE 连接
- 分段推送回答内容
- 前端实时展示
- 输出结束标记
- 异常中断处理
- 流式完成后统一落库

### 9.6 自然语言导航模块

功能说明：

- 将操作型自然语言转换为前端可执行动作

功能点：

- 输入预处理
- 规则匹配
- 参数提取
- 结构化 JSON 结果返回
- 前端跳转或自动筛选
- 导航记录保存

结果字段建议：

- `action`
- `target_page`
- `keyword`
- `params`

一期支持范围：

- 打开竞赛列表页
- 搜索人工智能类竞赛

### 9.7 记录与统计模块

功能说明：

- 记录用户核心行为
- 形成基础统计能力

记录范围：

- 检索记录
- 咨询记录
- 导航记录

统计范围：

- 咨询总次数
- 热门问题关键词
- 热门竞赛类别
- 常用导航指令
- 活跃时间段

展示范围：

- 学生可看自己的历史记录
- 学生可看全局热门统计
- 管理员可看全站统计

### 9.8 后台管理模块

功能说明：

- 管理员维护竞赛分类和竞赛信息

竞赛分类管理：

- 新增分类
- 编辑分类
- 删除分类
- 分类排序

竞赛信息管理：

- 新增竞赛
- 编辑竞赛
- 删除竞赛
- 查看列表

竞赛信息字段要求：

- 竞赛名称
- 所属分类
- 赛事级别
- 适合专业
- 适合年级
- 报名开始时间
- 报名结束时间
- 官网链接
- 竞赛简介
- 参赛要求

## 10. 数据需求

### 10.1 核心数据表

- `user_info`
- `competition_category`
- `competition_info`
- `search_log`
- `consult_record`
- `navigation_command`
- `statistic_summary`

### 10.2 关键数据说明

`user_info`

- 存储学生和管理员账号信息
- 包括用户名、密码、专业、年级、角色、状态等字段

`competition_category`

- 存储竞赛分类

`competition_info`

- 存储竞赛主体信息

`search_log`

- 记录检索条件、结果数和检索时间

`consult_record`

- 记录问题、答案、咨询类型、返回方式和咨询时间

`navigation_command`

- 记录用户输入的导航指令、识别结果和执行结果

`statistic_summary`

- 存储汇总统计结果或统计快照

## 12. 非功能需求

### 12.1 性能要求

- 首页和列表页常规查询应快速返回
- 智能咨询普通问答应在可接受时间内返回结果
- SSE 模式应能够持续稳定输出内容

### 12.2 可用性要求

- 页面结构清晰，适合答辩演示
- 用户主要操作不超过 3 次点击即可到达核心功能
- 空数据和异常情况应有明确提示

### 12.3 可维护性要求

- 前后端分离清晰
- 后端模块职责明确，便于扩展
- 数据字段命名清晰，便于后续论文撰写与答辩说明

### 12.4 安全性要求

- 登录接口具备基本身份校验
- 管理员页面需做权限隔离
- 不在前端暴露敏感配置

## 13. 验收标准

### 13.1 学生端验收

- 能成功登录系统
- 能在首页查看竞赛列表
- 能按专业、赛事级别、报名状态筛选竞赛
- 能进入竞赛详情页查看完整信息
- 能在智能咨询页发起普通问答
- 能在智能咨询页发起 SSE 流式问答
- 能进行至少一轮连续追问
- 能使用“打开竞赛列表页”和“搜索人工智能类竞赛”两条导航指令
- 能在记录与统计页看到自己的记录与全局热门统计

### 13.2 管理端验收

- 管理员能成功登录后台
- 能新增、编辑、删除竞赛分类
- 能新增、编辑、删除竞赛信息
- 修改后的竞赛信息能在前台被正常检索和查看

## 14. 风险与注意事项

- 智能咨询质量受模型接口稳定性影响
- 若竞赛基础数据不足，咨询质量会明显下降
- 自然语言导航一期规则较少，需控制答辩时演示话术
- 前端页面由其他工具生成后，需预留一定时间进行结构调整和接口对接

## 15. 开发建议

建议按以下顺序推进：

1. 数据库表结构设计
2. 后台竞赛分类与竞赛信息管理
3. 前台竞赛列表与详情
4. 智能咨询与 SSE
5. 自然语言导航
6. 记录与统计
7. 联调与答辩演示优化
